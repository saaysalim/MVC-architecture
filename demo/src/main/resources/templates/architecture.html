System-level view of request flow and responsibility boundaries

1) Layer Responsibilities

Controller Layer

  • Maps routes using @GetMapping
  • Creates model attributes for templates
  • Exposes REST endpoints under /api

View Layer

  • Renders pages with Thymeleaf templates
  • Presents architecture and workflow pages
  • Provides navigation among all pages

Model/Data Layer

  • Current demo uses lightweight model attributes
  • Ready to extend with entities/services/repos
  • Holds business data and validation logic

2) Request Processing Flow

Step 1: Browser requests a route such as / or /architecture.
Step 2: HelloController selects the target template.
Step 3: Attributes are attached to the model when needed.
Step 4: Spring MVC resolves and renders the HTML template.
Step 5: Client receives rendered page and can navigate to Git or GitHub Home.
