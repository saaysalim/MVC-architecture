🏗️ MVC Design Architecture

System-level view of request flow and responsibility boundaries

🔧 MVC System Architecture Diagram

📊 1) Component Responsibilities

🎯 Controller/Servlet

  • Entry point for all requests
  • Routes to appropriate handler
  • Passes data to Model
  • Directs response to View

📱 View Layer

  • Thymeleaf HTML templates
  • Displays data from Model
  • User interface rendering
  • Client interaction

💾 Model Layer

  • Java Bean objects
  • Business logic
  • Data validation
  • Database operations

🗄️ Database

  • Persistent data storage
  • Data retrieval
  • Transaction handling
  • Entity persistence

🔄 2) Complete Request-Response Workflow

1. Client Request: Browser sends HTTP GET request (e.g., GET /architecture)
2. DispatcherServlet Receives: Controller intercepts the request and routes it
3. Handler Method Executes: @GetMapping method in HelloController runs
4. Model Population: Controller queries Model layer for business logic and data
5. Database Access: Model fetches data from Database via JPA/Repository
6. Data Return: Database sends results back to Model
7. View Resolution: Controller passes data attributes to View (template name)
8. Template Rendering: Thymeleaf renders HTML with injected model data
9. Response Sent: Rendered HTML sent back to client browser
10. Page Display: Browser displays the rendered page with navigation links

🏢 3) Container Architecture

Container Definition: A Spring Boot application runs inside a container that manages components, lifecycle, and HTTP handling.
Spring Context: IoC (Inversion of Control) container manages beans, dependencies, and component lifecycle.
Embedded Tomcat: Acts as the servlet container within Spring Boot for HTTP request processing.
Separation of Concerns: All three layers (Controller, View, Model) work together within the managed container environment.
